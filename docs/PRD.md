## 一、产品概述

### 1.1 产品定位

**一句话定位**: 话题交流与网盘资源分享融合的综合社区平台

**产品愿景**: 打造一个让用户既能深度讨论感兴趣的话题，又能便捷分享和获取优质资源的一站式社区

### 1.2 核心价值主张

| 价值点 | 描述 | 用户收益 |
|-------|------|---------|
| 一站式体验 | 话题讨论与资源分享深度整合 | 无需多平台切换 |
| 高质量内容 | 信用体系+内容质量加权 | 获取有价值信息 |
| 便捷资源获取 | 网盘资源集中展示与分类 | 快速找到所需资源 |
| 活跃社区氛围 | 激励机制+社区自治 | 良好的互动体验 |

### 1.3 差异化竞争策略

- **vs Reddit/V2EX**: 增加资源分享模块，满足用户"边讨论边分享"的需求
- **vs 传统网盘论坛**: 增强话题讨论功能，提升社区互动质量
- **vs 豆瓣小组**: 聚焦综合性话题+资源，覆盖更广泛用户群体

### 1.4 目标用户

**核心用户群体**:

| 用户类型 | 年龄 | 特征 | 核心需求 |
|---------|-----|------|---------|
| 技术爱好者 | 18-35岁 | 程序员/工程师/技术从业者 | 技术讨论、资源共享、行业资讯 |
| 学习者 | 16-30岁 | 学生/职场新人 | 学习资料、教程分享、经验交流 |
| 兴趣社群用户 | 18-40岁 | 追求归属感的群体 | 找到同好、深度交流、持续互动 |
| 内容创作者 | 20-40岁 | 有表达欲望的用户 | 发布内容、获得反馈、积累影响力 |

### 2.2 用户系统

#### 2.2.1 注册与登录

**功能描述**: 支持多种方式注册和登录

| 功能项 | 说明 | 优先级 |
|-------|------|-------|
| 邮箱注册 | 邮箱验证码注册 | P0 |
| 用户名登录 | 用户名+密码登录 | P0 |
| 手机号注册 | 手机验证码注册（国内） | P1 |
| 第三方登录 | 微信/GitHub/Google OAuth | P2 |
| 找回密码 | 邮箱验证重置密码 | P0 |

**注册流程**:
1. 用户填写注册信息（用户名、邮箱、密码）
2. 发送验证邮件/短信
3. 用户验证后激活账户
4. 引导完善个人资料

**账户字段**:
- 用户名（唯一，4-20字符）
- 昵称（可重复）
- 邮箱（唯一）
- 手机号（可选）
- 头像
- 个人简介
- 注册时间
- 最后登录时间

#### 2.2.2 个人中心

**功能描述**: 用户管理个人信息和内容

| 功能项 | 说明 | 优先级 |
|-------|------|-------|
| 个人资料 | 编辑昵称、头像、简介等 | P0 |
| 我的帖子 | 查看自己发布的话题 | P0 |
| 我的回复 | 查看自己的回复历史 | P0 |
| 我的资源 | 管理发布的资源 | P0 |
| 我的收藏 | 收藏的话题和资源 | P0 |
| 消息通知 | 系统通知、回复提醒 | P0 |
| 账户设置 | 密码修改、绑定信息 | P0 |

#### 2.2.3 等级与积分系统

**功能描述**: 激励用户贡献的积分体系

**积分规则**:

| 行为 | 积分 | 每日上限 |
|-----|-----|---------|
| 注册 | +50 | - |
| 每日登录 | +2 | 2 |
| 发布话题 | +5 | 50 |
| 发布回复 | +2 | 20 |
| 被点赞 | +1 | 无 |
| 被收藏 | +2 | 无 |
| 发布资源 | +10 | 100 |
| 资源被下载 | +1 | 无 |

**等级体系**:

| 等级 | 名称 | 所需积分 | 特权 |
|-----|-----|---------|-----|
| 1 | 新手上路 | 0 | 基础功能 |
| 2 | 初级会员 | 100 | 发布资源 |
| 3 | 中级会员 | 500 | 自定义头像框 |
| 4 | 高级会员 | 2000 | 专属徽章 |
| 5 | 资深会员 | 5000 | 申请版主 |
| 6 | 核心会员 | 10000 | 专属特权 |

#### 2.2.4 关注与粉丝

**功能描述**: 用户社交关系

- 关注用户：可关注其他用户
- 粉丝列表：查看关注自己的人
- 关注动态：查看关注用户的最新动态
- 私信功能：与关注用户私信（P1）

### 2.3 话题讨论模块

#### 2.3.1 板块与标签

**功能描述**: 内容分类组织

**板块系统**:
- 预设板块：技术、生活、学习、娱乐、资源等
- 板块描述：每个板块有简介和版规
- 板块版主：管理员可任命版主管理板块
- 板块订阅：用户可订阅感兴趣的板块

**标签系统**:
- 话题可添加1-5个标签
- 标签云展示热门标签
- 点击标签查看相关话题
- 标签管理：管理员可创建/合并/删除标签

**预设板块**:

| 板块名称 | 描述 | 示例标签 |
|---------|------|---------|
| 技术交流 | 技术讨论、经验分享 | 前端、后端、AI、运维 |
| 学习资料 | 学习资源、教程分享 | 编程、语言、设计 |
| 资源分享 | 网盘资源、工具分享 | 软件、电子书、教程 |
| 生活杂谈 | 生活话题、趣事分享 | 美食、旅行、电影 |
| 求助问答 | 问题求助、经验求助 | 怎么办、求推荐 |

#### 2.3.2 话题发布

**功能描述**: 用户发布话题内容

**话题字段**:
- 标题（必填，5-100字符）
- 正文（必填，支持Markdown）
- 板块（必选）
- 标签（可选，1-5个）
- 附件图片（可选，最多9张）
- 资源链接（可选，关联资源帖）

**发布规则**:
- 新用户（等级1）每日发帖上限3篇
- 等级2及以上无限制
- 敏感词自动过滤
- 支持草稿保存

#### 2.3.3 回复与互动

**功能描述**: 话题下的互动功能

**回复功能**:
- 楼层式回复
- 支持@用户
- 支持引用回复
- 支持Markdown
- 支持图片上传

**互动功能**:

| 功能 | 说明 | 优先级 |
|-----|------|-------|
| 点赞 | 对话题/回复点赞 | P0 |
| 踩 | 对话题/回复踩（匿名） | P2 |
| 收藏 | 收藏话题到个人收藏夹 | P0 |
| 分享 | 分享到社交平台 | P1 |
| 举报 | 举报违规内容 | P0 |
| 楼中楼 | 回复的回复 | P1 |

#### 2.3.4 话题展示与排序

**展示方式**:
- 列表视图（默认）
- 卡片视图（可选）
- 精华/置顶标识

**排序方式**:
- 最新发布
- 最新回复
- 热门（综合浏览、回复、点赞）
- 精华

### 2.4 资源分享模块

#### 2.4.1 资源发布

**功能描述**: 用户分享网盘资源

**资源字段**:
- 资源名称（必填）
- 资源描述（必填）
- 资源分类（必选）
- 网盘链接（必填）
- 提取码（可选）
- 封面图片（可选）
- 标签（可选）
- 积分要求（下载所需积分，默认0）

**资源分类**:

| 一级分类 | 二级分类 |
|---------|---------|
| 软件工具 | 开发工具、设计工具、效率工具、系统工具 |
| 学习教程 | 编程教程、语言学习、职业培训、考试资料 |
| 电子书 | 技术书籍、小说文学、社科经管、杂志期刊 |
| 影视资源 | 电影、剧集、纪录片、动漫 |
| 音频资源 | 音乐、播客、有声书 |
| 其他 | 模板素材、游戏、其他 |

**发布规则**:
- 等级2及以上可发布资源
- 资源审核：发布后进入审核队列（可选）
- 版权声明：发布时需同意版权协议

#### 2.4.2 资源浏览与搜索

**功能描述**: 发现和查找资源

**浏览方式**:
- 分类浏览
- 标签筛选
- 排序：最新、热门、下载量

**搜索功能**:
- 关键词搜索
- 分类筛选
- 时间筛选
- 排序方式

#### 2.4.3 资源下载

**功能描述**: 获取资源链接

**下载流程**:
1. 用户点击资源详情
2. 查看资源信息和评价
3. 点击"获取链接"
4. 如需积分，扣除积分后显示链接
5. 用户复制链接到网盘下载

**积分机制**:
- 免费资源：直接获取
- 积分资源：下载扣除发布者设定积分
- 发布者获得下载积分的80%
- 平台收取20%作为手续费

#### 2.4.4 资源评价

**功能描述**: 对资源进行评价反馈

**评价方式**:
- 评分：1-5星
- 评论：文字评价
- 标签：有效/失效/有毒等

**评价展示**:
- 资源平均评分
- 最新评价列表
- 有效/失效标记

#### 2.5.3 热门排行

**榜单类型**:

### 3.3 关键交互说明

#### 3.3.1 点赞交互

- 点击点赞按钮，数字+1，按钮变为高亮状态
- 再次点击取消点赞，数字-1，恢复默认状态
- 动画效果：点赞时轻微放大弹跳

#### 3.3.2 发帖交互

- 点击发布按钮，打开编辑器模态框或跳转发帖页
- 支持实时预览Markdown
- 支持拖拽上传图片
- 发布成功后跳转到话题详情页

#### 3.3.3 搜索交互

- 输入时显示搜索建议
- 按回车或点击搜索按钮执行搜索
- 搜索结果页支持筛选和排序

## 四、非功能需求

### 4.1 性能要求

| 指标 | 目标值 | 说明 |
|-----|-------|------|
| 页面加载时间 | < 2秒 | 首页及列表页 |
| 接口响应时间 | < 500ms | 95%请求 |
| 并发用户数 | 1000+ | 支持同时在线 |
| 数据库查询 | < 100ms | 95%查询 |
| 静态资源加载 | CDN加速 | 图片、JS、CSS |

### 4.2 安全要求

| 安全项 | 要求 |
|-------|------|
| 密码存储 | bcrypt加密，强度10+ |
| 传输加密 | HTTPS全站加密 |
| XSS防护 | 输入过滤、输出转义 |
| CSRF防护 | Token验证 |

### 5.1 MVP核心功能

**第一阶段必须实现**:

| 模块 | 功能 | 优先级 |
|-----|------|-------|

| 搜索 | 全站搜索 | P0 |

| 功能 | 原因 | 后续计划 |
|-----|------|---------|

| 私信功能 | 非核心功能 | 第二阶段 |
| 推荐系统(个性化) | 算法复杂 | 第二阶段 |
| 积分交易 | 需要完善的积分体系 | 第三阶段 |
| 移动App | 先保证Web体验 | 第三阶段 |

### 5.2 MVP验收标准

1. 用户可以注册、登录、管理个人信息
2. 用户可以浏览板块、发布话题、回复、点赞、收藏
3. 用户可以发布资源、浏览资源、获取下载链接
4. 支持全站搜索话题和资源
5. 管理员可以管理内容、用户和系统配置
6. 页面加载流畅，移动端体验良好
